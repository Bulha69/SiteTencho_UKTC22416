# Metal Shop — уебсайт за курсова работа

Уебсайт за магазин за metal облекло, разработен с **PHP + MySQL**.
Съдържа регистрация със снимка, логин с хеширани пароли, магазин,
управление на продукти (CRUD: добавяне/изтриване) и профилна страница.

## Структура на проекта

```
metal-shop/
├── config.php              # връзка с базата данни + помощни функции
├── database.sql             # SQL схема + примерни данни
├── index.php                 # начална страница
├── register.php             # регистрация (с качване на снимка)
├── login.php                 # вход (password_verify - хеш проверка)
├── logout.php                # изход от системата
├── products.php              # публичен магазин (списък с продукти)
├── manage_products.php       # управление: добавяне + изтриване (изисква логин)
├── profile.php                # профилна страница на потребителя
├── includes/
│   ├── header.php             # обща навигация
│   └── footer.php             # общ футър
├── assets/
│   ├── style.css               # цялостен дизайн (тъмна metal естетика)
│   ├── default-avatar.png
│   └── default-product.png
└── uploads/
    ├── avatars/                # качени профилни снимки
    ├── products/                # качени снимки на продукти
    └── .htaccess                  # защита - забранява изпълнение на скриптове
```

## Инсталация (локално, напр. с XAMPP / WAMP / MAMP)

1. **Копирай папката** `metal-shop` в `htdocs` (XAMPP) или `www` (WAMP).

2. **Създай базата данни:**
   - Отвори phpMyAdmin
   - Импортирай `database.sql` (създава базата `metal_shop` с таблиците
     `users` и `products`, плюс примерни продукти)

3. **Настрой връзката с базата данни** в `config.php`:
   ```php
   define('DB_USER', 'root');   // твоят MySQL потребител
   define('DB_PASS', '');       // твоята MySQL парола
   ```

4. **Провери права за писане** върху `uploads/avatars/` и `uploads/products/`
   (трябва PHP да може да записва файлове там).

5. Отвори в браузър: `http://localhost/metal-shop/`

## Какво да обясниш при защита на проекта

### 1. Сигурност на пароли (хеширане)
- При регистрация: `password_hash($password, PASSWORD_BCRYPT)` —
  паролата НИКОГА не се пази в чист текст.
- При логин: `password_verify($password, $hash)` — сравнява въведената
  парола с хеша, без да го декриптира (bcrypt е еднопосочен алгоритъм).

### 2. Защита от SQL Injection
- Всички заявки към базата данни използват **prepared statements** (PDO
  с `?` placeholders), никога директно вмъкване на потребителски вход в SQL.

### 3. Защита от XSS
- Всеки изход на потребителски данни минава през функцията `e()`
  (`htmlspecialchars`), за да се избегне инжектиране на HTML/JS код.

### 4. Качване на файлове — сигурност
- Проверка на разширение (whitelist: jpg, jpeg, png, gif, webp).
- Проверка чрез `getimagesize()`, че файлът наистина е изображение
  (не само преименуван .php файл).
- Ограничение на размера (3MB).
- Генериране на случайно име на файла (`random_bytes`), за да не може
  потребител да презапише чужд файл или да познае пътя.
- `.htaccess` в `uploads/`, който забранява изпълнение на скриптове,
  дори ако злонамерен файл бъде качен.

### 5. Контрол на достъп
- `requireLogin()` пази страниците `manage_products.php` и `profile.php`
  — недостъпни без активна сесия.
- Сесиите се управляват с вградения PHP `session_start()`.

### 6. База данни — релационен модел
- `users` (потребители) и `products` (продукти) са свързани чрез
  `created_by` (FOREIGN KEY) — показва кой потребител е добавил продукта.

## Възможни разширения (ако се изисква нещо повече)

- Редакция (Edit) на продукт, не само добавяне/изтриване.
- Роли (admin/user) — само admin да може да изтрива продукти.
- Количка за пазаруване и поръчки.
- Търсене и филтриране на продукти по категория.
